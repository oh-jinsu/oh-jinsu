`Rust`의 소유권(Ownership)이란 단순하게 말해서 메모리 해제에 대한 책임을 의미한다. 원칙은 단순하다. `Rust`는 변수를 스코프가 끝날 때 버린다.

[공식문서](https://doc.rust-lang.org)는 다음과 같이 정확하게 적어 두었다.
> - Each value in Rust has a variable that's called its owner.
- There can only be one owner at a time.
- When the owner goes out of scope, the value will be dropped.

예를 들면 아래의 함수는 스코프의 마지막에서 변수 a에 할당한 메모리를 해제한다.
```rust
fn do_something(a: String) {
  // do something
}
```

따라서 위의 함수를 호출하는 다음의 코드는 컴파일되지 않는다.
```rust
fn main() {
  let a = String::from("Hello, world!");

  do_something(a);

  do_something(a);
}
```

왜냐하면 첫번째로 호출한 함수 `do_something`이 변수 a의 메모리를 해제하는 책임을 넘겨 받았기 때문이다. rust에서 일어나는 소유권의 이동(move)이다. 이후의 시점에 a의 메모리는 이미 해제되었을 것이다. 함수의 인자로 넘겨 줄 수 없다.

하지만 다음에서 보듯이 참조자(reference)를 받는 함수는 변수 a에 할당한 메모리를 해제해야 하는 책임을 지니지 않는다.
```rust
fn do_something(a: &String) {
  // do something
}
```

빌림(borrowing)이라고 한다. 따라서 다음의 코드는 컴파일된다.
```rust
fn main() {
  let a = String::from("Hello, world!");

  do_something(&a);

  do_something(&a);
}
```
